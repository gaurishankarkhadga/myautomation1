# CreatorHub Backend - Complete Code Documentation & Architecture Map

This document serves as a comprehensive map of the backend services, models, and routes. Use this to quickly navigate the codebase without having to read every file individually.

## 1. Entry Point & Core Setup
- **`backend/server.js`**
  - **Purpose:** The main application entry point.
  - **Responsibilities:**
    - Initializes Express app and Mongoose database connection.
    - Sets up CORS and custom JSON/Raw-Body parsing (specifically parsing `req.rawBody` for Instagram Webhook signature verification).
    - Imports and binds API route modules (`/api/instagram`, `/api/youtube`, `/api/chat`, `/api/biolinks`, `/api/assets`).
    - Configures generic static serving for uploaded assets (`/uploads`).
    - **CRON JOBS:** Contains background recurring tasks like the `viral_tag_scheduled_reply` worker and the `Comment-to-DM` auto-expiration worker (checking time limits and comment count limits).

## 2. API Routes (`backend/route/`)
- **`instaautomationapi.js`**
  - **Purpose:** Core Instagram integration logic.
  - **Responsibilities:** OAuth connection flow, webhook receiver (`/webhook`), processing comments and DMs sent from Instagram in real-time, executing standalone requests.
- **`youtubeapi.js`**
  - **Purpose:** YouTube integration logic.
  - **Responsibilities:** OAuth flow and YouTube webhooks/comment interactions.
- **`chatapi.js`**
  - **Purpose:** Interface for the ChatHub.
  - **Responsibilities:** Connects the frontend chat UI to `chatService.js`. Handles posting messages, retrieving chat history, getting automation status counting (`/active-count/:userId`), etc.
- **`assetsapi.js`**
  - **Purpose:** Management of Creator Assets.
  - **Responsibilities:** CRUD routes for uploaded assets (files, links, courses) that the AI can distribute in DMs.
- **`biolinkapi.js`**
  - **Purpose:** Public profiles management.
  - **Responsibilities:** Create, Read, Update, Delete operations for the Biolink feature. 

## 3. Core Business Services (`backend/service/`)
- **`chatService.js`**
  - **Purpose:** The AI "Brain" for parsing user intent from Chat.
  - **Responsibilities:** Passes raw messages to Gemini to extract structured intents (e.g., `enable_comment_to_dm`). Dynamically loads and executes specific intent files from the `handlers/` directory.
- **`aiService.js`**
  - **Purpose:** Core Instagram natural language AI functions.
  - **Responsibilities:** 
    - Analyzes creator persona (reading past posts/replies to mimic style).
    - Generates Smart Replies based on Persona.
    - Matches user DM intent with available `CreatorAssets`.
    - Online research mapping for creators.
- **`inboxTriageService.js`**
  - **Purpose:** DM qualification.
  - **Responsibilities:** Categorizes DMs (Collaboration, Support, Fan Mail, Spam). Extracts brand details and auto-drafts responses/rate cards for potential sponsorships.
- **`mediaUtils.js`**
  - **Purpose:** Media formatting & filtering.
  - **Responsibilities:** Fetches a creator's media library and maps it against their `CreatorPreference` (e.g., Target latest reel, all reels, specific post).
- **`geminiClient.js`**
  - **Purpose:** Base implementation wrapping the Google Gemini SDK for fallback/retry capabilities.
- **`brandDealService.js`**
  - **Purpose:** Brand Deal discovering algorithm.
  - **Responsibilities:** Summarizes profile metrics and simulates identifying matching brand opportunities.
- **`viralTagService.js`**
  - **Purpose:** Schedules and controls viral tagging features (time-delayed engagements).
  - **Responsibilities:** Saves delayed events to be picked up by the CRON jobs in `server.js`.

## 4. Chat Intent Handlers (`backend/service/handlers/`)
Files here are dynamically loaded by `chatService.js` to process specific commands from the user:
- **`commentToDm.handler.js`**: Reconfigures the DB for Comment-to-DM triggers (parsing keywords, media targets, custom messages, time limits).
- **`commentAutoReply.handler.js`**: Configures global post comment auto replies.
- **`dmAutoReply.handler.js`**: Configures DM auto replies.
- **`status.handler.js`**: Generates the automation dashboard stats, active automation previews, and comment/DM transaction logs.
- **`morningBriefing.handler.js`**: Aggregates the last 24 hours of automation stats into an AI-narrated summary.
- **`brandDeals.handler.js`**: Manages triggering brand deal discovery and listing them in the chat.
- **`profile.handler.js`**: Triggers persona analysis and fetches profile summaries.
- **`preferences.handler.js`**: Modifies global user preferences (target media, time limits).

## 5. MongoDB Models (`backend/model/`)
- **`Instaautomation.js`**: A large collection containing heavily interconnected schemas:
  - `Token`: OAuth access tokens mapped to internal `userId`.
  - `AutoReplySetting` / `DmAutoReplySetting` / `CommentToDmSetting`: User-specific automation configurations.
  - `AutoReplyLog` / `DmAutoReplyLog`: Transaction records for successfully sent or failed replies.
  - `Conversation` / `Message`: Tracking active conversations (mostly for inbox triage context).
  - `WebhookEvent`: A queue structure for delayed processing (like viral tags).
- **`CreatorPersona.js`**: Stores the analyzed communication style and online research context.
- **`CreatorAsset.js`**: The actual digital products (links, pdfs) uploaded by the user to be used in AI interactions.
- **`CreatorPreference.js`**: Global user state settings like time limits or what type of content to target.
- **`BrandDeal.js` / `Campaign.js` / `DealApplication.js`**: Managing the brand deal lifecycle and UI representation.
- **`ChatHistory.js`**: Linear conversation logs for the AI Chat Hub interface.
- **`BioLink.js`**: Stores link-in-bio widget layouts, themes, and public URLs.
- **`YoutubeAutomation.js`**: Mirror schemas for YouTube data logic.

## Common Modification Guides
- **To add a new AI chat command:** Create a new file in `service/handlers/`, define the `intent` string, map the logic, and update the prompt map in `chatService.js` so Gemini knows the intent exists.
- **To fix a webhook bug:** Open `route/instaautomationapi.js` and trace the `scheduleAutoReply`, `scheduleDMAutoReply`, or `sendPrivateReply` logic blocks. Also verify `resolveUserIdMapping` if there are authentication/connection discrepancies between webhook metadata and DB tokens.
- **To modify the AI's personality:** Check `service/aiService.js` under the `generateSmartReply` or `generateSmartDMReply` functions where prompt blocks inject the `CreatorPersona.js` data.

## 6. Additional / Legacy Backend Files (`backend/Biolinksfiles/`)
- Contains legacy or experimental features from early builds (`biolink.js`, `instagramService.js`, `appp.js`, `database-init.js`). Not actively bound in `server.js` but kept for reference and migrations.
- Modern biolink routes are safely integrated into `/api/biolinks` via `backend/route/biolinkapi.js`.

## 7. Frontend Integration Touchpoints
- If modifying an API endpoint in the backend, remember to update the corresponding frontend calls in `frontend/src/components/` and `frontend/src/pages/`.
- The main chat interactions fire from `frontend/src/components/ChatHub.jsx`, which communicates directly with `backend/route/chatapi.js` (`POST /api/chat/message`).
