# Execution Plan: The Autonomous AI Agency

This comprehensive plan covers the entire blueprint outlined in the `technical_strategy_v2.md` as well as the newly requested multi-asset execution and ultra-flexible custom intent parsing.

## User Review Required

> [!WARNING]
> This plan will fundamentally change the core logic of the application. The system will shift from a "dumb" keyword responder to an "always-on" AI employee. Please review the implications in Phase 1 carefully.

## Proposed Changes

---

### Phase 1: The Core Pivot (Autonomous Routing)

Remove the restrictive visual dashboard toggles from the core webhook processor to allow the AI to handle all intent naturally.

#### [MODIFY] `backend/route/instaautomationapi.js`
- **Logic Overhaul:** In `scheduleDMAutoReply`, remove the strict `if(!settings.enabled) return;` block. 
- **New Logic:** If the user has active `CreatorAssets` uploaded, the AI is continuously authorized to scan incoming DMs for explicit purchase intent and reply autonomously based on semantic matching. 

#### [MODIFY] `frontend/src/components/AnalyticsDashboard.jsx` (and related routing)
- Downgrade the prominence of traditional Visual Dashboards. Shift their status to an "Advanced Settings" menu to reduce beginner overwhelm.

---

### Phase 2: Ultra-Flexible Custom Rules (Extra 1)

Ensure the AI can accept complex, multi-layered constraints from the user via chat, beyond the predefined logic limits.

#### [MODIFY] `backend/model/Instaautomation.js`
- Add `customInstructions: [String]` to the `dmAutoReplySettingSchema`.

#### [MODIFY] `backend/service/chatService.js` (The Brain)
- Inject a generic `custom_automation_instruction` into the Gemini intent parser.
- **Why:** If the user says: *"If anyone asks about nutrition, reply in Hindi but send the exact English ebook link"* – Gemini will capture this complex command.
- Create a new handler (`customRule.handler.js`) to parse this intent and append it to the DB schema array.

#### [MODIFY] `backend/service/aiService.js`
- Inside `generateSmartDMReply`, dynamically inject the user's `customInstructions` array directly into the system prompt overriding default conversational behaviors.

---

### Phase 3: Multi-Asset AI Execution (Extra 2)

Ensure the AI can handle users asking for completely distinct products/assets in a single DM simultaneously.

#### [MODIFY] `backend/route/instaautomationapi.js`
- In `scheduleDMAutoReply`, when `matchResult.matchedAssets` returns more than one asset (e.g. 2 different ebooks).
- We will update the `sendDirectMessage` timeout logic looping over `matchedAssets`, dispatching multiple Instagram messages sequentially, fully fulfilling the user's request.

---

### Phase 4: Deal Negotiator UI Integration

The `inboxTriageService.js` actively drafts $1,000+ negotiation deals in the background natively. We must expose this to the Creator elegantly.

#### [MODIFY] `frontend/src/components/ChatHub.jsx`
- Add a new "Draft Review" UI card component.
- When `ChatHub` mounts or receives data containing `negotiationData` from conversations, it will inject an AI message: *"🔔 Brand Alert! [Brand Name] reached out via DM. Based on your followers, I drafted a reply asking for [Suggested Rate]. Want to read my draft?"*

---

### Phase 5: The "Morning Briefing" Onboarding

Turn the AI into an active manager that reports actions to the boss (the Creator) rather than just waiting.

#### [MODIFY] `backend/service/chatService.js`
- Create a `get_morning_briefing` intent. This parses the `DmAutoReplyLog` and `WebhookEvent` database over the last 24 hours.
- Summarizes the data into one sentence: *"Good Morning! While you slept, I sent 14 workout ebook links via DM and successfully hid 2 spam comments on your latest Reel."*
- Trigger this event automatically on the first ChatHub page load of the day.
